# Vercel 배포 가이드

## 사전 준비

### 1. Supabase 설정
1. [Supabase](https://supabase.com)에서 프로젝트 생성
2. SQL Editor에서 다음 스키마 실행:

```sql
create table if not exists users (
  id bigint generated by default as identity primary key,
  email text unique not null,
  password text not null,
  is_admin boolean default false,
  is_approved boolean default false,
  created_at timestamp with time zone default timezone('utc'::text, now())
);

-- Optional: Enable RLS
alter table users enable row level security;
create policy "Allow all access for demo" on users for all using (true) with check (true);
```

3. Admin 계정 생성 (SQL Editor에서):
```sql
INSERT INTO users (email, password, is_admin, is_approved) 
VALUES (
  'henry@nextsecurities.com',
  '$2b$12$...', -- generate_password_hash('admin123') 결과값
  true,
  true
);
```

### 2. Vercel 배포 방법

#### 방법 1: Vercel CLI 사용

```bash
# Vercel CLI 설치
npm i -g vercel

# 프로젝트 디렉토리에서
cd /Users/henry/Documents/growth-projection

# 배포
vercel

# 프로덕션 배포
vercel --prod
```

#### 방법 2: GitHub 연동 (권장)

1. GitHub에 저장소 생성 및 푸시:
```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin <your-github-repo-url>
git push -u origin main
```

2. [Vercel Dashboard](https://vercel.com/dashboard)에서:
   - "Add New Project" 클릭
   - GitHub 저장소 선택
   - 프로젝트 설정:
     - Framework Preset: Other
     - Root Directory: ./
   - Environment Variables 추가:
     - `SUPABASE_URL`: Supabase 프로젝트 URL
     - `SUPABASE_KEY`: Supabase anon key
     - `FLASK_SECRET_KEY`: 랜덤 시크릿 키 (예: `openssl rand -hex 32`)
   - "Deploy" 클릭

### 3. 환경 변수 설정

Vercel Dashboard의 프로젝트 Settings > Environment Variables에서 다음 변수들을 추가:

- `SUPABASE_URL`: Supabase 프로젝트 URL
- `SUPABASE_KEY`: Supabase anon/public key
- `FLASK_SECRET_KEY`: Flask 세션 암호화 키 (랜덤 문자열)

### 4. Admin 계정 생성

배포 후 Supabase SQL Editor에서 admin 계정을 생성하거나, 
signup 후 admin이 직접 승인할 수 있습니다.
