# Vercel 배포 가이드

## 현재 상태
- ✅ 코드는 SQLite (로컬)와 Supabase (Vercel) 모두 지원
- ✅ 환경 변수에 따라 자동으로 선택
- ✅ 기본값 업데이트 완료 (LT180=88.52, LT30=14.75, decay-rate=54)

## 배포 단계

### 1. GitHub에 푸시

터미널에서 다음 명령어 실행:

```bash
cd /Users/henry/Documents/growth-projection
git push origin main
```

인증이 필요한 경우:
- GitHub Personal Access Token 사용
- 또는 SSH 키 설정

### 2. Vercel 환경 변수 설정

[Vercel Dashboard](https://vercel.com/dashboard)에서:

1. 프로젝트 선택 (`growth-projection`)
2. **Settings** > **Environment Variables** 이동
3. 다음 변수들 추가:

#### 필수 환경 변수:

- **SUPABASE_URL**
  - 값: Supabase 프로젝트의 URL
  - 예: `https://xxxxx.supabase.co`
  - 모든 환경에 적용 (Production, Preview, Development)

- **SUPABASE_KEY**
  - 값: Supabase 프로젝트의 `anon` 또는 `service_role` key
  - 예: `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`
  - 모든 환경에 적용

- **FLASK_SECRET_KEY**
  - 값: Flask 세션 암호화를 위한 랜덤 문자열
  - 생성 방법: `openssl rand -hex 32`
  - 예: `a1b2c3d4e5f6...`
  - 모든 환경에 적용

### 3. Supabase 데이터베이스 설정

1. [Supabase Dashboard](https://app.supabase.com) 접속
2. 프로젝트 선택 또는 새로 생성
3. **SQL Editor**에서 다음 스키마 실행:

```sql
-- Users 테이블 생성
create table if not exists users (
  id bigint generated by default as identity primary key,
  email text unique not null,
  password text not null,
  is_admin boolean default false,
  is_approved boolean default false,
  created_at timestamp with time zone default timezone('utc'::text, now())
);

-- RLS 활성화 (선택사항)
alter table users enable row level security;
create policy "Allow all access for demo" on users for all using (true) with check (true);
```

4. Admin 계정 생성:

먼저 Python으로 비밀번호 해시 생성:
```python
from werkzeug.security import generate_password_hash
print(generate_password_hash('admin123', method='pbkdf2:sha256'))
```

그 다음 SQL로 삽입:
```sql
INSERT INTO users (email, password, is_admin, is_approved) 
VALUES (
  'henry@nextsecurities.com',
  'pbkdf2:sha256:600000$...',  -- 위에서 생성한 해시
  true,
  true
);
```

### 4. 배포 확인

1. GitHub에 푸시하면 Vercel이 자동으로 배포 시작
2. [Vercel Dashboard](https://vercel.com/dashboard)에서 배포 상태 확인
3. 배포 완료 후 `https://growth-projection.vercel.app` 접속
4. 로그인 테스트:
   - Email: `henry@nextsecurities.com`
   - Password: `admin123`

## 문제 해결

### 배포 실패 시
- Vercel Dashboard의 **Deployments** 탭에서 로그 확인
- 환경 변수가 제대로 설정되었는지 확인
- Supabase 연결이 정상인지 확인

### 로그인 실패 시
- Supabase에 admin 계정이 생성되었는지 확인
- 비밀번호 해시가 올바른지 확인
- Supabase RLS 정책이 올바르게 설정되었는지 확인
